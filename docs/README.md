## 工作流程说明

本系统采用简化版敏捷开发流程，包含五个状态：

```
规划中 → 进行中 → 待审查 → 待处理修复 → 已完成
```

### 1. 规划中 (01-规划中)

存放待处理的任务。

**何时使用**：
- 新需求提出时
- 想法/灵感记录时
- 从 AI 对话中提取待办事项时

**操作**：
1. 复制 `template.md` 到本文件夹
2. 填写任务信息
3. 文件名格式：`TASK-YYYYMMDD-序号-简短描述.md`

### 2. 进行中 (02-进行中)

存放正在执行的任务。

**何时使用**：
- 开始处理某个规划中的任务时

**操作**：
1. 将任务文档从 `01-规划中` 移动到 `02-进行中`
2. 更新文档中的【状态】为"进行中"
3. 记录开始时间

### 3. 待审查 (03-待审查)

存放已完成待用户审查的任务。

**何时使用**：
- 任务代码编写完成并已提交 Git 后

**操作**：
1. 完成任务编码
2. **提交 Git**（重要！代码进仓库，文档留在本地）
3. 在任务文档底部追加【实现记录】部分：
   - 修改的文件列表
   - 实现说明
   - 提交信息
4. 将文档移动到 `03-待审查`
5. 更新【状态】为"待审查"

### 4. 待处理修复 (04-待处理修复)

存放待处理的 Bug、技术债务和优化任务。

**何时使用**：
- 发现需要延后处理的 Bug
- 识别出技术债务
- 性能优化需求
- 从"待审查"退回的返工任务

**操作**：
1. 在本文档夹创建问题描述文档
2. 标注优先级和影响范围
3. 安排时间处理，移动到"进行中"

**流转方向**：
```
规划中 → 待处理修复（直接创建的 Bug 任务）
进行中 → 待处理修复（发现的新问题）
待审查 → 待处理修复（审查不通过，返工）
待处理修复 → 进行中 → 待审查 → 已完成
```

### 5. 已完成 (05-已完成)

存放已审查通过的任务。

**何时使用**：
- 用户审查确认无误后

**操作**：
1. 审查通过后将文档移动到 `05-已完成`
2. 更新【状态】为"已完成"
3. 可选：记录审查意见

## 任务文档模板

每个任务使用 `template.md` 模板创建，包含：

- 基本信息（ID、标题、状态、优先级）
- 任务描述
- 验收标准
- 实现记录（完成后填写）

---

## Git 提交规范

当任务进入【待审查】状态时，需要提交 Git：

```bash
# 1. 查看修改的文件
git status

# 2. 添加相关文件（不包含 docs/）
git add src/... tests/...

# 3. 提交（参考任务文档中的标题）
git commit -m "feat: 实现用户登录功能

- 添加登录表单验证
- 集成 JWT 认证
- 添加错误处理

Refs: TASK-20240306-001"

# 4. 推送
git push
```

**注意**：`docs/` 文件夹已被 `.gitignore` 忽略，不会被提交。

## 与 AI 协作的最佳实践

### 1. 需求澄清阶段
- 在 `01-规划中` 创建任务文档
- 与 AI 讨论需求细节，记录在文档中
- 明确验收标准

**AI 应该阅读 [agent.md](./agent.md)** 了解项目结构和开发规范。

### 2. 开发阶段
- 将任务移到 `02-进行中`
- AI 生成代码时，记录关键决策点

### 3. 代码审查阶段
- 审查 AI 生成的代码
- 确认无误后提交 Git
- 填写【实现记录】
- 移到 `03-待审查`

### 4. 问题处理
- 如果发现 Bug 需要延后修复，创建文档放入 `04-待处理修复`
- 如果审查不通过，返回 `02-进行中` 或移到 `04-待处理修复`

### 5. 归档阶段
- 最终确认后移到 `05-已完成`
- 定期清理已完成任务（可选）

## 文件命名规范

```
TASK-YYYYMMDD-序号-简短描述.md

示例：
TASK-20240306-001-实现用户登录.md
TASK-20240306-002-添加数据验证.md
TASK-20240307-001-修复内存泄漏.md
```

---

## 优先级标识

在文件名或文档中使用：

- **P0** - 紧急阻塞
- **P1** - 高优先级
- **P2** - 正常优先级
- **P3** - 低优先级/可选

示例：`TASK-20240306-001-P1-实现用户登录.md`

---

## 快速开始

1. 复制 `template.md` 到 `01-规划中`
2. 重命名为合适的文件名
3. 填写任务详情
4. 开始开发时移到 `02-进行中`
5. 完成后提交 Git，更新文档，移到 `03-待审查`
6. 如有问题需要返工，移到 `04-待处理修复`
7. 审查通过后移到 `05-已完成`
